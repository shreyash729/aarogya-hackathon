Predictive Patient Risk Models Documentation

Overview

The Predictive Patient Risk Models app forecasts hospital readmission risks using a machine learning model trained on a Kaggle dataset. It calculates risk scores and identifies contributing factors, aiding healthcare providers in patient management.

Problem Statement

Develop tools to forecast hospital readmissions and calculate patient risk scores to support proactive healthcare interventions.

Tech Stack

  • Backend: Python, Flask
  • Machine Learning: Scikit-learn (RandomForestClassifier), SMOTE for handling imbalanced data
  • Data Processing: Pandas, NumPy
  • Visualization: Plotly Express for risk and feature importance charts
  • Frontend: HTML, CSS, JavaScript, Jinja2 templates
  • Dataset: Kaggle Hospital Readmissions (link)

Dataset Details

The model was trained on the Kaggle Hospital Readmissions dataset, containing 25,000 patient records with the following features:

  • age: Age bracket of the patient
  • time_in_hospital: Days (1-14)
  • n_procedures: Number of procedures performed
  • n_lab_procedures: Number of lab procedures
  • n_medications: Number of medications administered
  • n_outpatient: Outpatient visits in the prior year
  • n_inpatient: Inpatient visits in the prior year
  • n_emergency: Emergency room visits in the prior year
  • medical_specialty: Specialty of the admitting physician
  • diag_1, diag_2, diag_3: Primary and secondary diagnoses
  • glucose_test, A1Ctest: Test results (high, normal, not performed)
  • change, diabetes_med: Diabetes medication changes and prescriptions
  • readmitted: Target variable ('yes' or 'no')

Feasibility

  • Predictive Power: The model provides actionable risk scores and feature importance, aiding clinical decision-making.
  • Interpretability: Visualizations (risk and importance charts) make results accessible to non-technical users.
  • Integration: Flask-based deployment allows easy integration into hospital systems.
  • Data Utilization: Leverages comprehensive Kaggle dataset, covering diverse patient features.

Implementation Details

The app uses a web form to collect patient data (age, diagnosis, hospital stay, etc.). The RandomForest model predicts readmission probability, converted to a risk score (0-100). Plotly generates bar charts for risk scores and feature importance. Results are displayed via Jinja2 templates.
